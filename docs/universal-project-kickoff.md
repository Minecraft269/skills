# universal-project-kickoff

通用型项目启动与能力发现规则 — 适用于任何类型项目的启动阶段，执行意图探测 + 能力推荐 + 强制六步流程。

已合并原 `proactive-skill-discovery` 的全部工具发现能力。

## 核心流程

### Step 0：意图探测
先问用户想做什么，根据回答分流：

| 意图 | 行为 |
|------|------|
| 🚀 启动新项目 | 追问语言/框架 → 强制六步流程 + 能力推荐 |
| 💻 开发新功能 | 技术栈确认 → 开发工具推荐 |
| 🔍 审查代码 | 技术栈确认 → 审查工具推荐 |
| 🐛 修复 Bug | 技术栈确认 → 调试工具推荐 |
| 🔧 探索工具 | 完整能力扫描 + 命令发现 |

### 强制六步流程（启动新项目）

| 步骤 | 内容 | 产出 |
|------|------|------|
| 1 | 澄清"为什么"与"是什么" | 项目一句话定义 + 成功标准 |
| 2 | 圈定边界 | MVP 范围、三重约束 |
| 3 | 快速风险摸底 | Top 3 风险 + B 计划 |
| 4 | 利益相关者对齐 | 核心圈/影响圈/外围圈 |
| 5 | 绘制路线图 | 3-5 个里程碑 + 验收标准 |
| 6 | 生成 CLAUDE.md | 调用 /init 固化成果 |

### 能力推荐

合并了原 proactive-skill-discovery 的发现引擎：
- 项目指纹扫描（25+ 种配置文件检测）
- 并行能力扫描（Skills + Plugins + 深度探索）
- 智能匹配评分 + 按意图过滤
- 交互式推荐 + 命令发现
- 全量导出（Markdown/JSON/纯文本）

## 适用场景

- 新项目启动
- 功能规划
- AI Agent 设计
- 项目计划审查
- 代码审查工具推荐
- 调试工具推荐
- 技能/插件/命令探索

## 核心原则

**先开枪，后瞄准，但开枪前得知道靶子大概在哪个方向。** 15 分钟内完成关键决策，避免返工。

## 相关技能

本技能属于 [minecraft269-skills](https://github.com/Minecraft269/skills) 插件包。当完整安装插件包时，本技能可与其他包内技能自动联动：

- 识别技术栈后提示安装相关 MCP Server
- CLAUDE.md 生成后提示扫描技术栈推荐匹配技能
- 被 PR 管理器在克隆陌生项目后触发，帮助新贡献者快速理解项目

独立安装本技能时，上述联动功能静默关闭，不影响核心流程。
